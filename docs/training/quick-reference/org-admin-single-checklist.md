# Quick Reference: Org Admin (Single + Network)

Accounts:

- `pastor@example.com` / `AeraDemo!2026` (single-location)
- `ng1001.orgadmin@aera.demo` / `AeraDemo!2026` (network)

## Situation 1: Daily Operations

### Single-location

- Open org overview
- Check unknown/unreached members first
- Identify top vulnerable households
- Record top 3 follow-up actions

### Network

- Open parent rollup (`NG-1001`)
- Identify highest-risk child location
- Switch into each child and validate differences
- Record top 3 network priorities

## Situation 2: Pre-Impact (T-72 to T-24)

### Single-location

- Open Members + Preparedness tabs
- Rank outreach by life-safety risk
- Confirm contact strategy for highest-risk households
- Review inventory constraints for next 24h

### Network

- Compare `CH-9921` and `NGO-5500` constraints
- Decide support order with rationale
- Send one baseline network directive
- Send one targeted child directive

## Situation 3: Impact Window

### Single-location

- Monitor incoming status signals
- Send clear local broadcast
- Re-prioritize outreach continuously
- Track unresolved critical cases with owner + deadline

### Network

- Monitor multi-location operations quality
- Re-target directives when local conditions diverge
- Escalate blocked high-risk nodes quickly
- Keep location-level priorities updated

## Situation 4: Recovery (+24 to +72)

### Single-location

- Classify households: resolved / in progress / blocked
- Update local recovery priorities
- Publish short leadership brief

### Network

- Classify locations: stable / delayed / blocked
- Publish command brief with support focus and checkpoint time

## Broadcast Standard

Every message should include:

- Who must act
- What action is required
- By when
- Where to report updates

## Do Not

- Broadcast generic directives when local conditions differ
- Ignore transport/medical vulnerability signals
- Treat inventory/readiness as static snapshots
- Leave critical households or locations without status ownership
