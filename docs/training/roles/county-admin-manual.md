# AERA Role Manual: County Admin

Role: `COUNTY_ADMIN`

## 1) Role Purpose

County Admins coordinate cross-organization response within county boundaries and remove bottlenecks between local operations and state escalation.

## 2) Core Responsibilities

- Monitor county-wide population and status signals
- Compare readiness and impact across organizations
- Route support to highest-need locations
- Escalate county constraints to state level with evidence

## 3) Hands-On Workflow

### Step A: County Rollup Review

1. Open county-level overview
2. Identify top-risk organizations or zones
3. Confirm data freshness before decisions

**Pass condition:** trainee identifies top risk clusters with rationale.

### Step B: Resource Coordination

1. Match county constraints to available support capacity
2. Assign response focus by urgency and reachability
3. Track unresolved queues to avoid silent backlog growth

**Pass condition:** priority actions are explicit and time-bound.

### Step C: Escalation Quality

1. Package escalation with quantitative impact evidence
2. State what county can handle vs what needs state intervention
3. Log handoff decisions for accountability

**Pass condition:** escalation is complete, concise, and decision-ready.

## 4) Common Mistakes to Prevent

- Escalating without validated county context
- Ignoring data recency and stale incident status
- Treating all local organizations as operationally identical

## 5) Training Completion Check

County Admin passes when they can:

- Run county prioritization without ambiguity
- Coordinate cross-org support decisions
- Deliver high-quality state escalation packages
