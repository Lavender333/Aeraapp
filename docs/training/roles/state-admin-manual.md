# AERA Role Manual: State Admin

Role: `STATE_ADMIN`

## 1) Role Purpose

State Admins manage multi-county situational awareness, allocate strategic support, and ensure statewide operational consistency during disaster lifecycles.

## 2) Core Responsibilities

- Maintain statewide risk and recovery picture
- Balance support allocation across counties
- Set escalation thresholds and response priorities
- Audit statewide operational consistency and reporting quality

## 3) Hands-On Workflow

### Step A: Statewide Situational Picture

1. Review county-level trends and outliers
2. Separate immediate life-safety risk from medium-term recovery drag
3. Confirm signal quality before policy-level directives

**Pass condition:** trainee identifies where state intervention changes outcomes most.

### Step B: Strategic Allocation

1. Prioritize high-impact county interventions
2. Issue directive priorities with clear rationale
3. Re-check downstream effects after allocation changes

**Pass condition:** allocation decisions are traceable and defensible.

### Step C: Governance and Oversight

1. Verify county reporting completeness
2. Flag repeated process failures for corrective action
3. Produce board/leadership-ready summary outputs

**Pass condition:** summary captures risk, actions, and expected next-state.

## 4) Common Mistakes to Prevent

- Over-centralizing decisions without local context
- Allocating support based on anecdote vs verified data
- Failing to monitor post-allocation effect

## 5) Training Completion Check

State Admin passes when they can:

- Prioritize interventions across counties
- Explain allocation logic clearly
- Produce concise statewide operational summary
