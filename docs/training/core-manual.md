# AERA Core Manual (All Users)

Date: 2026-02-26

## 1) Learning Goals

By the end of this manual, users can:

- Sign in and complete required setup
- Understand role-based access boundaries
- Submit and interpret help/status information
- Maintain preparedness data accurately
- Follow consent and privacy expectations

## 2) Core Concepts

- **Community Code**: ties people to a trusted church/community node.
- **Preparedness Data**: household and vulnerability details used for emergency planning.
- **Help Flow**: structured incident intake replacing ad-hoc messaging during crises.
- **Role Boundaries**: each role sees only what it needs for safe operations.

## 3) Universal User Journey

1. Open app and authenticate
2. Complete identity + home + preparedness setup
3. Confirm consent for preparedness planning
4. Use dashboard as daily operating home
5. During incidents, complete help flow with accurate updates

## 4) Required Data Quality Standards

- Use real, reachable phone numbers for emergency contacts
- Keep home address and ZIP accurate for dispatch planning
- Keep household roster current (dependents, elderly, special needs)
- Update mobility/transport/medical constraints when conditions change

## 5) Privacy & Consent Talking Points (Trainer Script)

- Data is for preparedness and emergency response coordination.
- Consent must be explicit and can be withdrawn.
- Encourage users to update data whenever household conditions change.
- Explain that incomplete data increases response delays.

## 6) Common Failure Patterns and Fixes

- **Invalid login**: hard refresh, clear site storage, retry account.
- **Missing org context**: confirm Community Code or org assignment.
- **Stale profile fields**: revisit Settings and save updates.
- **Confusion on role access**: verify role before troubleshooting view access.

## 7) Completion Check (All Users)

A user passes core training if they can:

- Log in successfully
- Complete or review profile and preparedness fields
- Explain what Community Code does
- Demonstrate where to submit incident help status
- State one privacy/consent safeguard correctly
